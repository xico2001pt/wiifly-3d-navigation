int WiimoteApi.Wiimote.ReadWiimoteData ( )

Reads and interprets data reported by the Wii Remote.

Returns
On success, > 0, < 0 on failure, 0 if nothing has been recieved.

Wii Remote reads function similarly to a Queue, in FIFO (first in, first out) order. For example, if two reports were sent since the last ReadWiimoteData() call, this call will only read and interpret the first of those two (and "pop" it off of the queue). So, in order to make sure you don't fall behind the Wiimote's update frequency, you can do something like this (in a game loop for example):

Wii Remote wiimote;
int ret;
do
{
ret = wiimote.ReadWiimoteData();
} while (ret > 0);

Requests the Wii Remote to report data from its internal registers.

Parameters
typeThe type of register you would like to read from
offsetThe starting offset of the block of data you would like to read
sizeThe size of the block of data you would like to read
ResponderThis will be called when the Wii Remote finishes reporting the requested data.
Returns
On success, > 0, <= 0 on failure.
See also
SendRegisterWriteRequest(RegisterType, int, byte[])
Warning
Do not attempt to read from the registers when another read is pending (that is, data is being recieved by the Wii Remote). If you attempt to do this, the new read request will be ignored.

Reading from the Wii Remote's internal registers can give important data not available through normal output reports. This can, for example, be used to read saved Mii data from the Wii Remote's EEPROM registers. It is also used by some of WiimoteApi's setup functions.

If you use this incorrectly (for example, if you attempt to read from an invalid block of data), Responder will not be called.

Attempts to write a block of data to the Wii Remote's internal registers.

Parameters
typeThe type of register you would like to write to
offsetThe starting offset of the block of data you would like to write
dataData to write to registers at offset. This must have a maximum length of 16.
Returns
On success, > 0, <= 0 on failure.
Warning
If data.Length > 16 the write request will be ignored.

Writing to the Wii Remote's internal registers allows you to access advanced functions of the remote, such as the speakers or the IR camera. It is used by some of WiimoteApi's setup functions (SetupIRCamera() for example).

If you use this incorrectly (for example, if you attempt to write to a read-only register) the Wii Remote handles this gracefully and nothing happens.

bool WiimoteApi.Wiimote.SetupIRCamera ( IRDataType  type = IRDataType.EXTENDED)

Performs a series of coperations to initialize the IR camera.

Parameters
typeThe IR Report type you want to use.
Returns
If all IR setup commands were successfully sent to the Wii Remote.

This performs the following steps in order to set up the IR camera:

  1. Enable IR Camera (Send 0x04 to Output Report 0x13)
  2. Enable IR Camera 2 (Send 0x04 to Output Report 0x1a)
  3. Write 0x08 to register 0xb00030
  4. Write Sensitivity Block 1 to registers at 0xb00000
  5. Write Sensitivity Block 2 to registers at 0xb0001a
  6. Write Mode Number to register 0xb00033
  7. Write 0x08 to register 0xb00030 (again)
  8. Update the Wii Remote's data reporting mode based on type

bool WiimoteApi.Wiimote.RumbleOn = false

Represents whether or not to turn on rumble when sending reports to the Wii Remote. This will only be applied when a data report is sent. That is, simply setting this flag will not instantly enable rumble.
