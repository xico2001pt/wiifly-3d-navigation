float [] WiimoteApi.ClassicControllerData.GetLeftStick01 ( )

Returns the left stick analog values in the range 0-1.

Warning
This does not take into account zero points or deadzones. Likewise it does not guaruntee that 0.5f is the zero point. You must do these calibrations yourself.
float [] WiimoteApi.ClassicControllerData.GetRightStick01 ( )

Returns the right stick analog values in the range 0-1.

Note
The Right stick has half of the precision of the left stick due to how the Wiimote reports data. The right stick is therefore better for less precise input.
Warning
This does not take into account zero points or deadzones. Likewise it does not guaruntee that 0.5f is the zero point. You must do these calibrations yourself.

ReadOnlyArray<byte> WiimoteApi.ClassicControllerData.rstick
get

Classic Controller right stick analog values. This is a size-2 array [X,Y] of RAW (unprocessed) stick data. These values are in the range 0-31 in both X and Y.

Note
The Right analog stick reports one less bit of precision than the left stick (the left stick is in the range 0-63 while the right is 0-31).
